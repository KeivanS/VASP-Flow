\section{Introduction}

The VASP Workflow Agent is an automated system for setting up and running VASP (Vienna Ab initio Simulation Package) calculations. It reads plain-text instructions and generates all necessary input files, job scripts, and analysis tools.

\subsection{Key Features}
\begin{itemize}
    \item \textbf{Plain text interface}: Write instructions in natural language
    \item \textbf{Automated workflow}: Generates complete calculation sequences
    \item \textbf{Flexible methods}: Supports PBEsol, R2SCAN, HSE06, SOC, GGA+U
    \item \textbf{Convergence testing}: Automated k-point and ENCUT convergence
    \item \textbf{Post-processing}: Analysis and plotting scripts
    \item \textbf{Wannierization}: Integration with Wannier90
    \item \textbf{Transport}: BoltzWann support
\end{itemize}

\subsection{Requirements}
\begin{itemize}
    \item VASP 5.4 or later
    \item Python 3.6+
    \item Linux environment
    \item Pseudopotential (POTCAR) files
\end{itemize}

\section{Installation and Setup}

\subsection{Directory Structure}
The workflow agent uses the following directory structure:

\begin{lstlisting}[language=bash]
vasp-workflow/
├── vasp-agent.py           # Main executable script
├── modules/                # Python modules
│   ├── instruction_parser.py
│   └── vasp_input_generator.py
├── examples/               # Example instruction files
│   ├── example1_basic.txt
│   └── example2_advanced.txt
├── projects/               # Your projects (created automatically)
│   └── project_name/
│       ├── instructions.txt
│       ├── POSCAR
│       └── generated/      # Generated calculations
└── manual.tex             # This manual
\end{lstlisting}

\subsection{First-Time Setup}

\textbf{Step 1: Set executable permissions}
\begin{lstlisting}[language=bash]
chmod +x vasp-agent.py
\end{lstlisting}

\textbf{Step 2: Configure POTCAR path}

Edit the POTCAR directory path in generated \texttt{make\_potcar.sh} scripts, or set it as an environment variable:
\begin{lstlisting}[language=bash]
export VASP_POTCAR_DIR="$HOME/path/to/your/potcar"
\end{lstlisting}

\textbf{Step 3: Test with example}
\begin{lstlisting}[language=bash]
./vasp-agent.py --project test_run \
                --instructions examples/example1_basic.txt \
                --poscar /path/to/your/POSCAR
\end{lstlisting}

\section{Writing Instructions}

Instructions are written in a plain text file with a simple format.

\subsection{Basic Structure}

\begin{lstlisting}[language=bash]
Project: [Your project name]

Methods: [Functional and special methods]
         [Additional methods]

Tasks: [Calculation types]
       [More tasks]

Convergence: [Convergence tests]

Goals: [Optional: what you want to study]
\end{lstlisting}

\subsection{Instruction Keywords}

\subsubsection{Project Line}
\begin{lstlisting}
Project: MoS2 electronic structure
\end{lstlisting}
Names your project. This will be the title in output files.

\subsubsection{Methods Section}

\textbf{Functionals:}
\begin{itemize}
    \item \texttt{PBE} (default)
    \item \texttt{PBEsol}
    \item \texttt{R2SCAN}
    \item \texttt{HSE06} or \texttt{HSE}
    \item \texttt{RVV10} (van der Waals)
\end{itemize}

\textbf{Spin-Orbit Coupling (SOC):}
\begin{lstlisting}
SOC with magnetization in z-direction
SOC with magnetization in x-direction
\end{lstlisting}

\textbf{GGA+U:}
\begin{lstlisting}
GGA+U with U=3.0 on Mo-d orbitals
GGA+U with U=5.0 on Ni-d orbitals
\end{lstlisting}

\subsubsection{Tasks Section}

Available tasks (in execution order):
\begin{enumerate}
    \item \textbf{Relaxation} or \textbf{Optimization}: Structure relaxation
    \item \textbf{SCF} or \textbf{Self-consistent}: Static calculation
    \item \textbf{Band structure} or \textbf{Bands}: Electronic band structure
    \item \textbf{DOS}: Density of states
    \item \textbf{Fat bands}: Orbital-projected band structure
    \item \textbf{Wannierization}: Wannier90 interface
    \item \textbf{Transport}: BoltzWann transport calculations
\end{enumerate}

\textbf{Example:}
\begin{lstlisting}
Tasks: Relaxation
       Band structure along G-M-K-G
       DOS (projected on Mo-d and S-p)
\end{lstlisting}

\subsubsection{K-path Specification}

For band structure calculations, specify the high-symmetry path:
\begin{lstlisting}
Band structure along G-X-M-G-R-X
Band structure along G-K-M-G
\end{lstlisting}

Common k-points:
\begin{itemize}
    \item \texttt{G}: Gamma point (0,0,0)
    \item \texttt{X}: (0.5,0,0)
    \item \texttt{M}: (0.5,0.5,0)
    \item \texttt{K}: (0.333,0.333,0)
    \item \texttt{R}: Various definitions depending on lattice
\end{itemize}

\subsubsection{Convergence Tests}

\textbf{K-point convergence:}
\begin{lstlisting}
Convergence: Test k-points 8x8 to 16x16
Convergence: Test k-points from 6x6x6 to 12x12x12
\end{lstlisting}

\textbf{ENCUT convergence:}
\begin{lstlisting}
Convergence: ENCUT 400-600 eV
Convergence: Test ENCUT 400-600 eV in steps of 50 eV
\end{lstlisting}

\subsubsection{DOS Projections}

Specify orbital projections for DOS:
\begin{lstlisting}
DOS (projected on Mo-d and S-p)
DOS with projections on Ni:d and O:p
\end{lstlisting}

\subsection{Complete Example}

\begin{lstlisting}[language=bash]
Project: Topological insulator Bi2Se3

Methods: PBEsol functional
         SOC with magnetization in z-direction

\section{Running Calculations}

\subsection{Basic Usage}

\textbf{Command syntax:}
\begin{lstlisting}[language=bash]
./vasp-agent.py --project PROJECT_NAME \
                --instructions INSTRUCTIONS_FILE \
                --poscar POSCAR_FILE
\end{lstlisting}

\textbf{Short options:}
\begin{lstlisting}[language=bash]
./vasp-agent.py -p PROJECT_NAME -i INSTRUCTIONS_FILE -s POSCAR_FILE
\end{lstlisting}

\subsection{Workflow Steps}

After running the agent, you'll find:

\begin{lstlisting}[language=bash]
projects/PROJECT_NAME/
├── instructions.txt        # Your instructions (copied)
├── POSCAR                 # Your structure (copied)
└── generated/             # Generated files
    ├── 01_relax/          # Relaxation calculation
    │   ├── INCAR
    │   ├── KPOINTS
    │   ├── POSCAR
    │   ├── make_potcar.sh
    │   └── run.sh
    ├── 02_scf/            # SCF calculation
    ├── 03_bands/          # Band structure
    ├── 04_dos/            # DOS
    ├── submit_all.sh      # Run all calculations
    └── analyze_all.sh     # Analyze results
\end{lstlisting}

\subsection{Execution Options}

\textbf{Option 1: Run all calculations sequentially}
\begin{lstlisting}[language=bash]
cd projects/PROJECT_NAME/generated
./submit_all.sh
\end{lstlisting}

\textbf{Option 2: Run calculations individually}
\begin{lstlisting}[language=bash]
cd projects/PROJECT_NAME/generated/01_relax
./run.sh

# After completion, run next calculation
cd ../02_scf
./run.sh

# And so on...
\end{lstlisting}

\textbf{Option 3: Submit to SLURM (for cluster computing)}

Modify the run.sh scripts to use SLURM:
\begin{lstlisting}[language=bash]
#!/bin/bash
#SBATCH --job-name=vasp_calc
#SBATCH --nodes=1
#SBATCH --ntasks=16
#SBATCH --time=24:00:00

module load vasp

srun vasp_std > vasp.out 2>&1
\end{lstlisting}

\section{Understanding Generated Files}

\subsection{INCAR Files}

The agent generates appropriate INCAR files for each calculation type.

\textbf{Relaxation INCAR:}
\begin{itemize}
    \item \texttt{IBRION = 2}: Ionic relaxation algorithm
    \item \texttt{ISIF = 3}: Relax ions and cell shape
    \item \texttt{NSW = 100}: Maximum ionic steps
    \item \texttt{EDIFFG = -0.01}: Force convergence criterion
\end{itemize}

\textbf{Band structure INCAR:}
\begin{itemize}
    \item \texttt{ICHARG = 11}: Non-self-consistent from CHGCAR
    \item \texttt{LORBIT = 11}: Projected DOS output
    \item \texttt{LWAVE = .FALSE.}: Don't write WAVECAR
\end{itemize}

\subsection{KPOINTS Files}

\textbf{Automatic mesh (relaxation, SCF, DOS):}
\begin{lstlisting}
Automatic mesh
0
Gamma
40
\end{lstlisting}
The number 40 controls mesh density (length parameter in Angstroms).

\textbf{Line mode (band structure):}
\begin{lstlisting}
Line-mode KPOINTS
40
Line-mode
rec
  0.0000   0.0000   0.0000  ! G
  0.5000   0.0000   0.0000  ! X

  0.5000   0.0000   0.0000  ! X
  0.5000   0.5000   0.0000  ! M
...
\end{lstlisting}

\subsection{Job Scripts}

Each calculation directory contains a \texttt{run.sh} script that:
\begin{enumerate}
    \item Creates POTCAR if needed
    \item Copies files from previous calculations (for bands/DOS)
    \item Runs VASP
    \item Checks for successful completion
\end{enumerate}

\section{Customization}

\subsection{Modifying POTCAR Generation}

Edit \texttt{make\_potcar.sh} in each calculation directory:
\begin{lstlisting}[language=bash]
#!/bin/bash
# Change this to your POTCAR directory
POTCAR_DIR="$HOME/vasp/potcar/PBE"

# Or use PAW_PBE, PAW_GGA, etc.
POTCAR_DIR="$HOME/vasp/potcar/PAW_PBE"
\end{lstlisting}

\subsection{Adjusting INCAR Parameters}

You can manually edit any INCAR file before running:
\begin{lstlisting}[language=bash]
cd projects/PROJECT_NAME/generated/01_relax
nano INCAR  # or vim, emacs, etc.
\end{lstlisting}

Common adjustments:
\begin{itemize}
    \item \texttt{ENCUT}: Plane wave cutoff energy
    \item \texttt{EDIFF}: Electronic convergence criterion
    \item \texttt{ISMEAR}: Smearing method
    \item \texttt{MAGMOM}: Initial magnetic moments
\end{itemize}

\subsection{Modifying K-point Paths}

For band structure calculations, edit the KPOINTS file to change the path or add more k-points between high-symmetry points:
\begin{lstlisting}
Line-mode KPOINTS
80              # More points = smoother bands
Line-mode
rec
  0.0000   0.0000   0.0000  ! G
  0.3333   0.3333   0.0000  ! K
\end{lstlisting}

\section{Analysis and Post-Processing}

\subsection{Built-in Analysis}

After calculations complete, run:
\begin{lstlisting}[language=bash]
cd projects/PROJECT_NAME/generated
./analyze_all.sh
\end{lstlisting}

This creates an \texttt{analysis/} directory with extracted data.

\subsection{Manual Data Extraction}

\textbf{Extract total energy:}
\begin{lstlisting}[language=bash]
grep "energy  without" OUTCAR | tail -1
\end{lstlisting}

\textbf{Extract band gap (for insulators):}
\begin{lstlisting}[language=bash]
grep "E-fermi" OUTCAR | tail -1
\end{lstlisting}

\textbf{Extract magnetic moment:}
\begin{lstlisting}[language=bash]
grep "magnetization (x)" OUTCAR | tail -1
\end{lstlisting}

\subsection{Plotting Results}

Use the template in \texttt{analysis/plot\_results.py}:
\begin{lstlisting}[language=python]
#!/usr/bin/env python3
import numpy as np
import matplotlib.pyplot as plt

# Read band structure from EIGENVAL
eigenval = np.loadtxt('../03_bands/EIGENVAL', skiprows=6)

# Plot bands
plt.figure(figsize=(8,6))
plt.plot(eigenval[:,0], eigenval[:,1], 'b-')
plt.xlabel('K-path')
plt.ylabel('Energy (eV)')
plt.title('Band Structure')
plt.savefig('bands.png', dpi=300)
\end{lstlisting}

\section{Advanced Features}

\subsection{Convergence Testing}

The agent generates convergence test scripts in:
\begin{lstlisting}
generated/00_convergence/
├── kpoints/
│   └── run_kpoint_convergence.sh
└── encut/
    └── run_encut_convergence.sh
\end{lstlisting}

These are templates that you should customize for your specific needs.

\subsection{Wannierization}

If your instructions include Wannierization, additional Wannier90 input files will be generated. You'll need to:

\begin{enumerate}
    \item Run a bands calculation with \texttt{LWANNIER90 = .TRUE.}
    \item Generate \texttt{wannier90.win} input file
    \item Run Wannier90: \texttt{wannier90.x wannier90}
    \item Interpolate bands: \texttt{wannier90.x wannier90 -pp}
\end{enumerate}

\subsection{Transport with BoltzWann}

For transport calculations:
\begin{enumerate}
    \item Complete Wannierization
    \item Obtain Wannier Hamiltonian
    \item Run BoltzWann to calculate transport coefficients
    \item Analyze Seebeck coefficient, electrical conductivity, etc.
\end{enumerate}

\section{Troubleshooting}

\subsection{Common Errors}

\textbf{POTCAR not found}
\begin{lstlisting}
ERROR: POTCAR not found for Mo at /path/to/potcar/Mo/POTCAR
\end{lstlisting}
\textbf{Solution:} Edit \texttt{make\_potcar.sh} with correct POTCAR directory path.

\textbf{VASP not found}
\begin{lstlisting}
bash: vasp_std: command not found
\end{lstlisting}
\textbf{Solution:} Ensure VASP is in your PATH or use full path in run.sh:
\begin{lstlisting}
/path/to/vasp/bin/vasp_std > vasp.out 2>&1
\end{lstlisting}

\textbf{Memory error}
\begin{lstlisting}
Error: insufficient memory
\end{lstlisting}
\textbf{Solution:} Reduce k-point mesh density or ENCUT, or use more computational resources.

\subsection{Checking Convergence}

Always check if calculations converged:
\begin{lstlisting}[language=bash]
grep "reached required accuracy" OUTCAR
\end{lstlisting}

If not converged, increase NSW or adjust convergence criteria.

\subsection{Getting Help}

Check log files:
\begin{itemize}
    \item \texttt{vasp.out}: VASP standard output
    \item \texttt{OUTCAR}: Detailed VASP output
    \item \texttt{OSZICAR}: Convergence information
\end{itemize}

\section{Best Practices}

\begin{enumerate}
    \item \textbf{Always review generated files} before running calculations
    \item \textbf{Start with convergence tests} before expensive calculations (HSE, SOC)
    \item \textbf{Use relaxed structures} for band structure and DOS calculations
    \item \textbf{Check magnetic moments} in OUTCAR for magnetic systems
    \item \textbf{Verify k-point paths} match your crystal symmetry
    \item \textbf{Keep instructions files} with your data for reproducibility
    \item \textbf{Back up results} regularly, especially CHGCAR and WAVECAR
\end{enumerate}

\section{FAQ}

\textbf{Q: Can I use this with other DFT codes?}

A: Currently only VASP is supported. However, the modular design makes it relatively easy to add other codes.

\textbf{Q: How do I add custom INCAR tags?}

A: Edit the generated INCAR files directly, or modify the template in \texttt{modules/vasp\_input\_generator.py}.

\textbf{Q: Can I restart failed calculations?}

A: Yes. Copy CONTCAR to POSCAR and WAVECAR/CHGCAR from the previous run, then restart.

\textbf{Q: How do I know which pseudopotentials to use?}

A: Generally use PAW\_PBE for most calculations. Use PAW\_LDA for LDA, and GW pseudopotentials for GW calculations.

\textbf{Q: Can this run on computing clusters?}

A: Yes. Modify the run.sh scripts to use your cluster's job scheduler (SLURM, PBS, etc.).
