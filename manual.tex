\section{Introduction}
% ============================================================

VASP Workflow is a framework for setting up, running, and analysing DFT
calculations with VASP.  Two interfaces are provided:

\begin{itemize}
  \item \textbf{Browser GUI} (\texttt{vasp-gui.py}) --- a Flask web
        application: configure calculations through a form, run steps from
        the browser, view band-structure and DOS plots in real time.
  \item \textbf{CLI agents} (\texttt{vasp-agent.py} for workstations,
        \texttt{vasp-agent-slurm.py} for SLURM clusters) --- read a
        plain-text \texttt{instructions.txt} file and generate all INCAR,
        KPOINTS, POSCAR, POTCAR, and job scripts automatically.
\end{itemize}

Both interfaces share the same \texttt{modules/} back-end.

\subsection{Key features}

\begin{itemize}
  \item Functionals: PBEsol, PBE, R2SCAN, HSE06, RVV10, LDA, AM05
  \item Spin-orbit coupling (SOC) and collinear spin (ISPIN=2)
  \item Magnetic materials: correct MAGMOM per atom; spin-resolved band
        and DOS plots (spin-up / spin-down)
  \item GGA+U (Dudarev, LDAUTYPE=2) per element and orbital
  \item Automatic k-path detection from POSCAR lattice vectors
  \item Convergence testing: ENCUT and k-mesh sweeps
  \item DOS with orbital projections via \texttt{sumo}
  \item Phonons via phonopy, Wannier90 interface, DFPT Born charges
  \item Smart KPAR/NCORE auto-defaults scaled with MPI task count
  \item SLURM workflow with dependency-chained job submission
\end{itemize}

\subsection{Requirements}

\begin{tabularx}{\linewidth}{@{}lX@{}}

VASP 5.4+ & \texttt{vasp\_std}, \texttt{vasp\_ncl} (SOC), optionally
            \texttt{vasp\_gam} \\

\section{Installation}
% ============================================================

\begin{lstlisting}[language=bash]
# 1. Clone the repository
git clone https://github.com/KeivanS/VASP-Flow.git
cd VASP-Flow

# 2. Install Python dependencies
pip install flask sumo

# 3. Optional: phonons and Wannier support
conda install -c conda-forge phonopy wannier90

# 4. First-time configuration (creates site.env from template)
make setup
nano site.env     # set VASP paths, POTCAR directory, MPI command
\end{lstlisting}

\subsection{site.env --- platform configuration}

\begin{lstlisting}
PYTHON          = python3
VASP_STD        = /path/to/vasp_std
VASP_NCL        = /path/to/vasp_ncl
VASP_GAM        = /path/to/vasp_gam       # optional
VASP_POTCAR_DIR = /path/to/potpaw_PBE
MPI_LAUNCH      = mpirun -np              # or: srun
MPI_NP          = 16
WANNIER90_X     = wannier90.x
\end{lstlisting}

\section{Browser GUI}
% ============================================================

\subsection{Starting the GUI}

\begin{lstlisting}[language=bash]
make run          # launches on http://localhost:5001
\end{lstlisting}

\textbf{On a SLURM cluster} --- run on the login node, open on your
laptop via SSH port-forwarding:

\begin{lstlisting}[language=bash]
# On your laptop:
ssh -L 5001:localhost:5001 username@cluster

# On the cluster login node:
make run

# Open http://localhost:5001 in your laptop's browser
\end{lstlisting}

\subsection{Three-tab workflow}

\begin{enumerate}
  \item \textbf{Setup} --- configure the project, then click
        \emph{Generate Workflow}.
  \item \textbf{Workflow} --- run each calculation step; live VASP
        output streams to the browser.
  \item \textbf{Results} --- band-structure and DOS plots; OUTCAR analysis.
\end{enumerate}

To return to a previous project use the \emph{Select project} dropdown.
\textbf{$\to$ Workflow} jumps to the run tab; \textbf{Edit \&
Regenerate} re-opens Setup with all settings pre-filled.

Normal execution order: \textbf{relaxation $\to$ SCF $\to$ bands $\to$

\end{notebox}

\subsection{Band structure options}

Leave the \emph{k-path} field blank to trigger \textbf{automatic
detection} from the POSCAR lattice vectors (Section~\ref{sec:kpath}).
Enter a path manually to override, e.g.\ \texttt{G-M-K-G}.

\subsection{DOS options}

Add projection rows to request orbital-resolved DOS:

\medskip
\noindent\hspace{1cm}Element: \texttt{Mo}\quad Orbitals: \textbf{d}
\hspace{1.5cm}
Element: \texttt{S}\quad Orbitals: \textbf{p}
\medskip

For spin-polarised calculations sumo plots spin-up positive and
spin-down mirrored below zero automatically.

\subsection{Workflow tab}

Each step shows three controls: \textbf{Run} (submit the step),
\textbf{Edit files} (open INCAR/KPOINTS/POSCAR in the browser; a
\texttt{.bak} is created on first edit), and a \textbf{status badge}.
\textbf{Run All Steps} chains every step and stops on failure.

\subsection{Results tab}

\begin{tabularx}{\linewidth}{@{}lX@{}}
\thead
Panel & Contents \\ \midrule
Band structure & Spin-polarised: blue = $\uparrow$, red dashed = $\downarrow$,
  same k-axis, $E_F = 0$.  Non-magnetic: single-colour sumo plot.
  If sumo fails (e.g.\ a pymatgen gap-detection crash) the GUI falls back to a
  built-in \texttt{vasprun.xml} plotter so a band plot still renders. \\
Cumulative DOS & Stacked filled-area by element.
  Spin-polarised: up positive, down mirrored. \\
Projected DOS & Orbital-resolved via sumo; spin-down negated
  automatically. \\
OUTCAR analysis & Energy, Fermi level, magnetic moment, forces. \\
\bottomrule
\end{tabularx}

\section{CLI Agents}
% ============================================================

\subsection{vasp-agent.py --- workstation}

\begin{lstlisting}[language=bash]
./vasp-agent.py
./vasp-agent.py -i my_inst.txt -s struct.vasp

cd ProjectName
./run_all.sh          # runs every step in sequence
./analyze.sh          # plots after all steps complete
\end{lstlisting}

\subsection{vasp-agent-slurm.py --- SLURM clusters}

\begin{lstlisting}[language=bash]
./vasp-agent-slurm.py                  # uses profiles/slurm.json
./vasp-agent-slurm.py --profile slurm

cd ProjectName
./submit_all.sh        # sbatch with dependency chaining
squeue -u $USER        # monitor jobs
./analyze.sh           # run locally after jobs finish
\end{lstlisting}

Two-phase workflow when convergence tests are requested:

\begin{lstlisting}[language=bash]
./submit_convergence.sh     # STEP 1 -- submit convergence jobs
# review results, then:
./submit_calculations.sh    # STEP 2 -- prompts ENCUT/k-mesh, submits
\end{lstlisting}

\subsection{Generated directory structure}

\begin{lstlisting}
ProjectName/
  instructions.txt
  POSCAR
  POTCAR               built once from $VASP_POTCAR_DIR
  01_relax/            INCAR  KPOINTS  POSCAR  POTCAR  run.sh
  02_scf/
  03_bands/
  04_dos/
  analysis/            plots generated by analyze.sh
  run_all.sh           workstation sequential runner
  submit_all.sh        SLURM job submission (dependency chained)
  analyze.sh           post-processing: plots + OUTCAR extraction
\end{lstlisting}

\section{High-Throughput Screening (Materials Project)}
\label{sec:highthroughput}
% ============================================================

\texttt{ht-mp-scf.py} drives a batch of SCF + ELF calculations straight
from Materials Project IDs.  For every \texttt{mp-id} it downloads the
\textbf{primitive} cell from MP, stages a per-material
\texttt{instructions.txt} (an SCF task plus an \texttt{INCAR scf:} block
setting \texttt{LELF = .TRUE.} for the electron localization function),
and writes a \texttt{runall.sh} that calls the chosen agent to build the
inputs and then runs or submits each job.  Each SCF uses a medium
\texttt{10 10 10} $\Gamma$-mesh by default; edit the \texttt{KMESH} line in
\texttt{\_ht\_inputs/<id>/instructions.txt} (or pass \texttt{-{}-kmesh}) to
change it.

\begin{lstlisting}[language=bash]
pip install pymatgen mp-api          # MP download + primitive standardisation
export MP_API_KEY=...                 # Materials Project API key
export VASP_POTCAR_DIR=...            # POTCAR library (used by the agent)

# one mp-id per line (blank lines and # comments ignored)
printf 'mp-149\nmp-2534\n' > highthrouput_list

./ht-mp-scf.py --agent local  --mpi 16        # workstation
./ht-mp-scf.py --agent slurm  --profile slurm # cluster (chained by default)

./runall.sh                           # build inputs + run/submit every job
\end{lstlisting}

The ELFCAR for each material is written to \texttt{<mp-id>/02\_scf/}.
A summary \texttt{\_ht\_inputs/elements\_Z.txt} tabulates every element in
the screening set with its atomic number $Z$ (from a built-in table --- no
MP query), plus a per-material breakdown.

\begin{tabularx}{\linewidth}{@{}lX@{}}
\thead
Option & Meaning \\ \midrule
\texttt{-{}-agent} & \texttt{local} (vasp-agent.py) or \texttt{slurm}
  (vasp-agent-slurm.py); prompts if omitted. \\
\texttt{-{}-list} & ID file; default \texttt{highthrouput\_list}. \\
\texttt{-{}-functional} & PBE (default), PBEsol, R2SCAN, \ldots \\
\texttt{-{}-mpi} & MPI tasks per job. \\
\texttt{-{}-encut} & Optional ENCUT override (eV). \\
\texttt{-{}-kmesh} & SCF $\Gamma$-mesh, e.g. \texttt{'8 8 8'} or \texttt{'12'}
  (cubic); default medium \texttt{10 10 10}. Per-material edits via the
  \texttt{KMESH} line in \texttt{instructions.txt}. \\
\texttt{-{}-chain} / \texttt{-{}-no-chain} & SLURM only; see below. \\
\bottomrule
\end{tabularx}

\subsection{Execution order}

\begin{tabularx}{\linewidth}{@{}lX@{}}
\thead
Mode & Behaviour \\ \midrule
local & \texttt{run\_all.sh} blocks --- materials run strictly
  one-at-a-time. \\
slurm \texttt{-{}-chain} (default) & Each material's SCF is submitted with
  \texttt{-{}-dependency=afterok} on the previous one, so the cluster runs
  them sequentially even though all are queued up front. \\
slurm \texttt{-{}-no-chain} & Materials are submitted independently and run
  whenever the scheduler allows. \\
\bottomrule
\end{tabularx}

\begin{notebox}
The driver fetches structures over the network (Materials Project), so
run it on a node with internet --- typically the login node.
\texttt{runall.sh} performs no downloads and can run on a compute node.
\end{notebox}

\section{instructions.txt Format}
\label{sec:instructions}
% ============================================================

\subsection{Example}

\begin{lstlisting}
Project: My material name

Methods: PBEsol functional
         collinear magnetization
         GGA+U with U=4.0 on Fe-d orbitals

Tasks: structure relaxation
       SCF calculation
       band structure
       DOS (projected on Fe-d and O-p)

Convergence: Test k-points 6x6x6 to 12x12x12
             ENCUT 400-600 eV

MAGMOM: 4.0
NSW: 100
EDIFFG: -0.01
MPI: 128
NODES: 2
NTASKS_PER_NODE: 64
WALLTIME: 12:00:00
\end{lstlisting}

The parser is case-insensitive and searches the entire file, so section
order is flexible.

\subsection{Keyword reference}

\begin{tabularx}{\linewidth}{@{}lX@{}}
\thead
Keyword & Meaning \\ \midrule
\texttt{Project:} & Project name; becomes the output folder. \\
\texttt{MAGMOM:} & Initial moment in $\mu_{\mathrm{B}}$/atom (collinear only).
  Written as \texttt{MAGMOM = N*value}. Default 2.0. \\
\texttt{ISIF:} & Relaxation type: 3=full, 2=ions only, 4=fixed volume. \\
\texttt{NSW:} & Max ionic steps. Default 100. \\
\texttt{EDIFFG:} & Force criterion in eV/\AA. Default $-0.01$. \\
\texttt{ENCUT:} & Manual plane-wave cutoff in eV. \\
\texttt{KMESH:} & Explicit Gamma k-mesh overriding the automatic density,
  e.g.\ \texttt{8 8 8} or \texttt{12} (cubic).  Applies to every automatic
  mesh (relax/SCF/DOS); for 2-D give the third value (\texttt{12 12 1}). \\
\texttt{PRESSURE:} & External pressure for constant-$P$ relaxation, e.g.\
  \texttt{10 GPa} or \texttt{50 kbar}.  Forces \texttt{ISIF=3},
  \texttt{IBRION=2}, \texttt{PSTRESS} (Section~\ref{sec:pressure}). \\
\texttt{INCAR: \ldots\ END\_INCAR} & Block of literal INCAR tags merged
  into the generated INCAR(s); optionally per-step
  (Section~\ref{sec:incar-raw}). \\
\texttt{NKPTS:} & k-points per band segment. Default 40. \\
\texttt{MPI:} & Total MPI tasks (nodes $\times$ cores/node). \\
\texttt{KPAR:} & Global k-point parallelisation. \\
\texttt{NCORE:} & Global cores-per-orbital. \\
\texttt{SCF\_KPAR:} \textit{etc.} &
  Per-step KPAR: also BANDS\_, DOS\_, RELAX\_, DFPT\_, PHONONS\_. \\
\texttt{SCF\_NCORE:} \textit{etc.} & Per-step NCORE; same prefixes. \\
\texttt{NODES:} & Compute nodes (SLURM). \\
\texttt{NTASKS\_PER\_NODE:} & MPI tasks per node (SLURM). \\
\texttt{PARTITION:} & SLURM partition. \\
\texttt{WALLTIME:} & Time limit, e.g.\ \texttt{12:00:00}. \\
\texttt{ACCOUNT:} & SLURM account / allocation. \\
\texttt{DFPT\_EDIFF:} & EDIFF for DFPT. Default \texttt{1E-8}. \\
\texttt{PHONONS\_DIM:} & Supercell, e.g.\ \texttt{2 2 2}. \\
\texttt{PHONONS\_MESH:} & q-mesh, e.g.\ \texttt{20 20 20}. \\
\texttt{PHONONS\_DISP:} & Displacement in \AA. Default 0.01. \\
\texttt{PHONONS\_BAND:} & High-symmetry path for phonon bands. \\
\texttt{PHONONS\_NAC:} & \texttt{FALSE} to disable NAC. \\
\texttt{WANNIER\_NUM\_WANN:} & Number of Wannier functions. \\
\texttt{WANNIER\_PROJ:} & Projections, e.g.\ \texttt{Ga:s;As:p}. \\
\texttt{WANNIER\_EWIN:} & Energy window, e.g.\ \texttt{-5.0 to 15.0}. \\
\bottomrule
\end{tabularx}

\section{Methods}
% ============================================================

\subsection{Functionals}

\begin{lstlisting}
Methods: PBEsol functional    # recommended for solids
Methods: R2SCAN functional    # meta-GGA
Methods: HSE06 functional     # hybrid -- more accurate but slower
Methods: RVV10 functional     # van der Waals
\end{lstlisting}

Default ENCUT values: PBE 400~eV, PBEsol 450~eV, R2SCAN 680~eV,
HSE06 400~eV, RVV10 450~eV.

\subsection{Spin-orbit coupling}

\begin{lstlisting}
Methods: SOC with magnetization in z-direction
\end{lstlisting}

Uses \texttt{vasp\_ncl}.  Sets \texttt{LSORBIT}, \texttt{LNONCOLLINEAR},
and \texttt{SAXIS}.

\subsection{Magnetic materials (collinear spin)}
\label{sec:magnetic}

\begin{lstlisting}
Methods: collinear magnetization
MAGMOM: 4.0
\end{lstlisting}

Sets \texttt{ISPIN = 2} and \texttt{MAGMOM = N * value} in every INCAR,
where $N$ is the atom count from the POSCAR.

\begin{tipbox}
Typical starting moments: Fe, Co $\approx$ 4--5~$\mu_{\mathrm{B}}$;
Ni $\approx$ 2~$\mu_{\mathrm{B}}$; Cr, Mn $\approx$ 3--4~$\mu_{\mathrm{B}}$;
rare earths $\approx$ 7~$\mu_{\mathrm{B}}$.
VASP converges the self-consistent value automatically.
\end{tipbox}

\textbf{Antiferromagnets} --- after generation, edit INCAR to alternate
signs:
\begin{lstlisting}
MAGMOM = 4.0 -4.0 4.0 -4.0    ! alternating per sublattice
\end{lstlisting}

\textbf{Spin-resolved plots} produced automatically after DOS/bands run:

\begin{itemize}
  \item \emph{Cumulative DOS} --- spin-up above the x-axis; spin-down
        mirrored below zero; elements colour-coded; solid = $\uparrow$,
        dashed = $\downarrow$.
  \item \emph{Band structure} --- spin-up in blue, spin-down in red
        dashed, same k-axis, $E_F = 0$.
  \item \emph{Projected DOS} --- sumo natively negates the spin-down
        channel in the orbital-resolved PDF.
\end{itemize}

Check the converged moment:
\begin{lstlisting}[language=bash]
grep "magnetization (x)" OUTCAR | tail -5
\end{lstlisting}

\subsection{GGA+U}
\label{sec:ggau}

\begin{lstlisting}
Methods: GGA+U with U=4.0 on Fe-d orbitals
         GGA+U with U=3.0 on Ni-d orbitals
         GGA+U with U=6.0 on Ce-f orbitals
\end{lstlisting}

Dudarev scheme (LDAUTYPE=2); $J = 0$ is set automatically.
Combine freely with \texttt{collinear magnetization}.

\section{Advanced INCAR Control}
\label{sec:incar}
% ============================================================

Two mechanisms let you go beyond the built-in keywords without editing
each generated INCAR by hand: \textbf{raw INCAR passthrough} (inject any
VASP tag directly) and \textbf{constant-pressure relaxation}.

\subsection{Raw INCAR passthrough}
\label{sec:incar-raw}

Place literal INCAR lines --- in ordinary VASP syntax --- inside an
\texttt{INCAR:} \ldots\ \texttt{END\_INCAR} block in your
\texttt{instructions.txt}.  They are merged into the generated INCAR(s):

\begin{lstlisting}
INCAR:                  # global block -- applies to every step
   LREAL  = Auto
   ALGO   = Fast
   NELMIN = 6
END_INCAR
\end{lstlisting}

A tag that the agent already writes is \textbf{overwritten in place}
(so \texttt{ALGO = Fast} above replaces the default \texttt{ALGO =
Normal}); any tag the agent does not normally emit is appended under a
clearly labelled \texttt{\# User INCAR overrides} section, so nothing is
silently lost.

To target a single step, name it on the header line.  Recognised steps
are \texttt{relax}, \texttt{scf}, \texttt{bands}, \texttt{dos},
\texttt{wannier}, \texttt{dfpt}, \texttt{phonons}:

\begin{lstlisting}
INCAR scf:              # applies only to 02_scf
   EDIFF = 1E-8
   SIGMA = 0.10
END_INCAR
\end{lstlisting}

\begin{notebox}
Precedence: a \emph{per-step} block beats the \emph{global} block, and
both beat the agent's generated defaults.  This is the recommended way to
set tags the parser has no dedicated keyword for (e.g.\ \texttt{LMAXMIX},
\texttt{ADDGRID}, \texttt{LVTOT}, \texttt{IVDW}, \texttt{AMIX}).
\end{notebox}

\subsection{Constant-pressure relaxation}
\label{sec:pressure}

A single \texttt{PRESSURE} line turns the relaxation into a
constant-pressure optimisation: the agent forces \texttt{ISIF = 3} (cell
shape + volume + ions) and \texttt{IBRION = 2}, and imposes the target
pressure through \texttt{PSTRESS}.

\begin{lstlisting}
PRESSURE: 10 GPa        # also accepted: "5 kbar", "constant pressure 8"
\end{lstlisting}

VASP's \texttt{PSTRESS} tag is in \textbf{kBar} (1~GPa $=$ 10~kBar).  The
agent converts automatically; if no unit is given, \textbf{GPa is
assumed}.  The resulting \texttt{01\_relax/INCAR} reads, e.g.:

\begin{lstlisting}
# Constant-pressure relaxation @ 10 GPa  (PSTRESS = 100 kBar)
IBRION = 2
ISIF   = 3
NSW    = 100
EDIFFG = -0.01
PSTRESS = 100   ! external pressure in kBar (10 GPa)
\end{lstlisting}

\begin{tipbox}
\texttt{PSTRESS} adds the $PV$ term so the relaxed cell sits at the
requested external pressure.  Use a tight \texttt{EDIFFG} and a
well-converged \texttt{ENCUT}; the Pulay stress error grows at high
pressure, so verify with a higher \texttt{ENCUT} before trusting the
equilibrium volume.
\end{tipbox}

\section{K-Path for Band Structure}
\label{sec:kpath}
% ============================================================

\subsection{Auto-detection (recommended)}

Omit the path in \texttt{instructions.txt} or leave the field blank in
the GUI:

\begin{lstlisting}
Tasks: band structure        # no "along ..." -> auto-detect from POSCAR
\end{lstlisting}

The generator reads POSCAR lattice vectors, computes cell lengths and
angles, and selects the standard path.

\begin{tabularx}{\linewidth}{@{}llX@{}}
\thead
Type & Detection criterion & Default path \\ \midrule
cF (FCC) & $a{=}b{=}c$, angles $\approx 60^\circ$
         & G-X-W-K-G-L-U-W \\
cI (BCC) & $a{=}b{=}c$, angles $\approx 109.5^\circ$
         & G-H-N-G-P-H \\
cP (cubic) & $a{=}b{=}c$, $90^\circ$
          & G-X-M-G-R-X \\
hP (hexagonal) & $a{\approx}b$, $\gamma{=}120^\circ$
              & G-M-K-G-A-L-H-A \\
tP (tetragonal) & $a{\approx}b{\neq}c$, $90^\circ$
               & G-X-M-G-Z-R-A-Z \\
oP (orthorhombic) & $90^\circ$, $a{\neq}b{\neq}c$
                 & G-X-S-Y-G-Z-U-R-T-Z \\
rP (rhombohedral) & $a{=}b{=}c$, equal angles
                 & G-T-F-G-L \\
mP (monoclinic) & $\alpha{=}\gamma{=}90^\circ$, $\beta{\neq}90^\circ$
               & G-Y-A-Z-G-B-D \\
\bottomrule
\end{tabularx}

\begin{notebox}
For \textbf{conventional} FCC or BCC cells (right angles, 4 or 2 atoms),
auto-detection returns the cubic (cP) path.
Use the \textbf{primitive} cell for an unfolded FCC/BCC band structure ---
the primitive FCC cell has angles $\approx 60^\circ$ and is recognised
automatically.
\end{notebox}

The detected type appears in the KPOINTS comment:
\begin{lstlisting}
Line-mode KPOINTS  [FCC -- auto-detected]
\end{lstlisting}

\section{Parallelization}

\subsection{Auto-defaults}

When KPAR/NCORE are not specified the agent computes from total tasks $N$:

\begin{tabularx}{\linewidth}{@{}llll@{}}
\thead
Step & KPAR & NCORE & Notes \\ \midrule
SCF / DOS / relax & $\lfloor\sqrt{N}\rfloor$ & $N/\text{KPAR}$ & \\
bands & 1 & $N$ & Required for line-mode \\
DFPT & 1 & $N$ & Required for IBRION=8 \\
phonons & 1 & $\lfloor\sqrt{N}\rfloor$ & \\
\bottomrule
\end{tabularx}

\begin{notebox}
KPAR $\times$ NCORE must divide $N$ evenly.  The agent rounds down
automatically to the nearest valid value.
\end{notebox}

\subsection{High-pressure relaxation with custom INCAR tags}
\label{sec:ex-pressure}

A constant-pressure cell relaxation at 10~GPa, combined with raw INCAR
passthrough to tighten convergence and add tags the parser has no
dedicated keyword for:

\begin{lstlisting}
Project: Silicon 10GPa

Methods: PBEsol functional

Tasks: structure relaxation
       SCF calculation
       band structure

PRESSURE: 10 GPa          # constant-P relax -> ISIF=3, IBRION=2, PSTRESS=100

# Tags applied to every step
INCAR:
   LREAL  = .FALSE.
   ADDGRID = .TRUE.
   LMAXMIX = 4
END_INCAR

# Tighter electronic convergence for the relaxation only
INCAR relax:
   EDIFF  = 1E-8
   EDIFFG = -0.005
END_INCAR

ENCUT: 600
MPI: 64
\end{lstlisting}

The \texttt{01\_relax/INCAR} is generated with \texttt{ISIF=3},
\texttt{PSTRESS=100}, the global tags appended, and \texttt{EDIFF}/%
\texttt{EDIFFG} overridden to the per-step values; \texttt{02\_scf} and
\texttt{03\_bands} receive only the global block.

\section{Troubleshooting}
% ============================================================

\begin{tabularx}{\linewidth}{@{}lX@{}}
\thead
Symptom & Solution \\ \midrule
POTCAR not found &
  Check \texttt{VASP\_POTCAR\_DIR} in \texttt{site.env} or GUI System
  Setup.  Use the POTCAR chooser for unusual variants
  (\texttt{Ga\_d}, \texttt{Fe\_sv}). \\[3pt]
\texttt{vasp\_std} not found &
  Set the full binary path in \texttt{site.env} or GUI System Setup. \\[3pt]
WARNING: may not have converged &
  Increase NSW (ionic) or NELM (electronic) in INCAR and re-run. \\[3pt]
Wrong bands ($W = \Gamma$) &
  Conventional cubic cell detected; use the \textit{primitive} cell
  for FCC/BCC so auto-detection picks cF/cI. \\[3pt]
Projected DOS missing &
  Check instructions.txt: \texttt{projected on} and \texttt{projections on}
  are both accepted.  Verify element names match POSCAR. \\[3pt]
KPAR error &
  KPAR $\times$ NCORE must divide total MPI tasks.  Use auto-defaults or
  set explicitly valid values. \\[3pt]
GUI unreachable &
  Check SSH tunnel: \texttt{ssh -L 5001:localhost:5001 user@cluster}. \\[3pt]
Project missing from dropdown &
  Folder must contain \texttt{POSCAR} + \texttt{settings.json} in the
  directory where \texttt{vasp-gui.py} was started. \\[3pt]
Band plot empty or not shown &
  Usually \texttt{sumo-bandplot} crashed on this band structure; the GUI now
  falls back to a built-in plotter automatically.  An empty plot also means a
  wrong $E_F$: the Fermi level is read from \texttt{04\_dos/OUTCAR} (then
  \texttt{02\_scf/OUTCAR}), since \texttt{03\_bands} eigenvalues share the
  NSCF/DOS energy reference. \\
\bottomrule
\end{tabularx}

\section*{Addendum: High-Throughput Quick Reference}
\addcontentsline{toc}{section}{Addendum: High-Throughput Quick Reference}
\label{sec:addendum-ht}
% ============================================================

Cheat-sheet for \texttt{ht-mp-scf.py} (full description in
Section~\ref{sec:highthroughput}).  For each Materials Project ID it
downloads the \textbf{primitive} cell, stages an SCF calculation that also
writes the electron localization function (\texttt{LELF = .TRUE.}
$\rightarrow$ \texttt{ELFCAR}) on a medium \texttt{10 10 10} $\Gamma$-mesh
(default; editable per material or via \texttt{-{}-kmesh}), and builds a
\texttt{runall.sh}.

\begin{lstlisting}[language=bash]
pip install pymatgen mp-api
export MP_API_KEY=...            # Materials Project key
export VASP_POTCAR_DIR=...       # POTCAR library (used by the agent)

printf 'mp-149\nmp-2534\n' > highthrouput_list   # one mp-id per line

./ht-mp-scf.py --agent local  --mpi 16           # workstation, sequential
./ht-mp-scf.py --agent slurm  --profile slurm    # cluster, afterok-chained
./ht-mp-scf.py --agent slurm  --no-chain         # cluster, independent

./runall.sh                      # build inputs + run/submit each SCF
\end{lstlisting}

\begin{tabularx}{\linewidth}{@{}lX@{}}
\thead
Output & Location \\ \midrule
ELFCAR (electron localization) & \texttt{<mp-id>/02\_scf/ELFCAR} \\
Staged inputs (POSCAR + instructions) & \texttt{\_ht\_inputs/<mp-id>/} \\
Element / $Z$ table & \texttt{\_ht\_inputs/elements\_Z.txt} \\
\bottomrule
\end{tabularx}

\begin{notebox}
\textbf{Transfer to a workstation:} only the project folder is needed
(code, \texttt{modules/}, \texttt{profiles/}); VASP binaries and the POTCAR
library already live there.  \texttt{ht-mp-scf.py} needs internet for the MP
download, so run it on the login node --- \texttt{runall.sh} does none.
\end{notebox}

\begin{notebox}
\textbf{If the download step errors with}
\texttt{'dict' object has no attribute 'number'}: that is an spglib /
pymatgen version mismatch.  Fix the root cause with
\texttt{pip install -U spglib}.  The driver also falls back to
\texttt{Structure.get\_primitive\_structure()} automatically, so a run is
never blocked by it.
\end{notebox}
