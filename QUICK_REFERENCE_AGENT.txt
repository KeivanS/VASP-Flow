┌─ BASIC USAGE ─────────────────────────────────────────────────────────────────┐
│ Workstation:                                                                  │
│   ./vasp-agent.py                        uses instructions.txt + POSCAR      │
│   ./vasp-agent.py -i my_inst.txt -s struct.vasp                              │
│   ./vasp-agent.py ... --incar "ALGO=Fast; NELMIN=6" --incar "scf:NBANDS=64"  │
│                       explicit INCAR tags from the command line (see below)  │
│                                                                               │
│ SLURM cluster (generates sbatch scripts + submit_all.sh):                    │
│   ./vasp-agent-slurm.py                  uses profiles/slurm.json            │
│   ./vasp-agent-slurm.py --profile slurm  explicit profile name               │
│                                                                               │
│ After generation:                                                             │
│   cd ProjectName                                                              │
│   ./run_all.sh               workstation: runs every step in sequence        │
│   ./submit_all.sh            SLURM: submits jobs with dependency chaining    │
│   ./analyze.sh               post-processing plots (run after all jobs done) │
└───────────────────────────────────────────────────────────────────────────────┘

┌─ DEFAULT HUBBARD U (automatic GGA+U) ─────────────────────────────────────────┐
│ When the POSCAR contains a tabulated d/f element TOGETHER WITH an             │
│ electronegative anion (O, F, S, Se, Te, Cl, Br, I), GGA+U (Dudarev) is      │
│ applied by DEFAULT                                                            │
│ with U/J looked up from  hubbard_u_defaults.csv  (repo root; element,        │
│ orbital, U, J + literature reference per row — edit/extend as needed).       │
│ Defaults = Materials Project fitted U where available, else the midpoint of  │
│ the linear-response (Cococcioni & de Gironcoli) range:                        │
│   d: V 3.25, Cr 3.7, Mn 3.9, Fe 5.3, Co 3.32, Ni 6.2, Cu 4.75, Mo 4.38,     │
│      W 6.2                                                                    │
│   f: La 10.3/J1.0, Ce 5.0, Pr 5.25, Nd 5.75, Sm 6.25, Eu 6.5, Gd 6.7/J0.7,  │
│      Tb-Yb 6.0, Th 3.0, U 4.5/J0.54, Pu 4.75                                 │
│ The CSV also lists the LR ranges + MP values per element. Calibrated for     │
│ PBE(-sol)/PAW — do NOT reuse with SCAN/R2SCAN or HSE06 without revalidation. │
│ LMAXMIX (4 d / 6 f) is set automatically with LDAU.                          │
│                                                                               │
│ Disable:   GGA_U: OFF        (or:  no GGA+U  /  without GGA+U)               │
│ Override:  Methods: GGA+U with U=4.0 on Fe-d    (explicit always wins)       │
│ No anion (e.g. LaBi, intermetallics) -> defaults NOT applied.                 │
└───────────────────────────────────────────────────────────────────────────────┘

┌─ EXPLICIT VASP KEYWORDS (any INCAR tag) ───────────────────────────────────────┐
│ Every VASP keyword can be specified explicitly — nothing is hard-wired.       │
│ A tag that already exists is overwritten in place; new tags are appended      │
│ under "User INCAR overrides". Precedence: --incar > per-step block > global   │
│ block > generated defaults.                                                    │
│                                                                               │
│ 1) Instructions file:                                                         │
│   INCAR:                 ← global block, applies to every step                │
│      LREAL = Auto                                                              │
│      ALGO  = Fast                                                              │
│      NELMIN = 6                                                                │
│   END_INCAR                                                                    │
│                                                                               │
│   INCAR scf:             ← per-step block (relax/scf/bands/dos/wannier/       │
│      EDIFF = 1E-8           dfpt/phonons/lobster)                              │
│   END_INCAR                                                                    │
│                                                                               │
│ 2) Command line (both agents; repeatable; ';' separates tags):                │
│   --incar "TAG=VAL; TAG2=VAL"          all steps                              │
│   --incar "scf:NBANDS=64"              one step only                          │
│                                                                               │
│ 3) GUI: every generated INCAR is editable per step (Workflow → Edit files).  │
│                                                                               │
│ Exceptions (re-imposed guards): ISYM=2 + KPAR=NCORE=1 in 06_dfpt;             │
│ symmetry-off ISYM in 08_lobster.                                              │
│ Note: full-line # comments in instructions files are ignored by the parser   │
│ (a task keyword inside a comment does not enable that task).                  │
└───────────────────────────────────────────────────────────────────────────────┘

│   Generated INCAR:  ISPIN = 2  /  MAGMOM = N*4.0  /  LORBIT = 11            │
│                                                                               │

│                                                                               │

│   Methods: SOC with magnetization in z-direction                              │
│   Methods: SOC with magnetization in x-direction                              │
│                                                                               │
│ Typical initial moments (μB):                                                 │
│   Fe, Co: 4–5    Ni: 2    Cr, Mn: 3–4    rare earths: 7                     │
│                                                                               │
│ Plots after DOS/bands run:                                                    │
│   DOS : per-element stacked + total DOS (black); up positive, down mirrored  │

┌─ TASKS ───────────────────────────────────────────────────────────────────────┐

┌─ K-PATH FOR BAND STRUCTURE ────────────────────────────────────────────────────┐
│ Omit "along …" → auto-detected from POSCAR:                                  │
│   Tasks: band structure                                                       │
│                                                                               │
│ Detection uses spglib (Setyawan–Curtarolo path for the real space group,     │
│ transformed into this POSCAR's reciprocal basis). If spglib/pymatgen is      │
│ missing, or for 2-D slabs, a geometric classifier picks these defaults:      │
│   cF  FCC primitive  (angles ≈ 60°)        G-X-W-K-G-L-U-W                  │
│   cI  BCC primitive  (angles ≈ 109.47°)    G-H-N-G-P-H                      │
│   cP  Cubic (right angles, a=b=c)          G-X-M-G-R-X                       │
│   hP  Hexagonal      (γ=120°)              G-M-K-G-A-L-H-A                   │
│        2D (is_2d flag)                     G-M-K-G                            │
│   tP  Tetragonal     (a≈b≠c, 90°)         G-X-M-G-Z-R-A-Z                   │
│   oP  Orthorhombic   (90°)                 G-X-S-Y-G-Z-U-R-T-Z               │
│   rP  Rhombohedral                         G-T-F-G-L                          │
│   mP  Monoclinic                           G-Y-A-Z-G-B-D                     │
│                                                                               │
│ Manual override (any crystal system):                                         │
│   Tasks: band structure along G-X-W-K-G-L                                    │
│                                                                               │
│ NOTE: conventional FCC/BCC cells (all 90°, 4/2 atoms) map to cP (cubic      │
│ path). Use the PRIMITIVE cell for unfolded FCC/BCC band structure — the       │
│ primitive cell is recognised automatically by its characteristic angles.     │
│                                                                               │
│ Available labels for manual paths:                                            │
│   G  X  M  K  R  Z  A  L  H  S  T  U  Y   generic / hexagonal               │
│   W  P  N                                   FCC/BCC primitive                 │
└───────────────────────────────────────────────────────────────────────────────┘

┌─ ELF BOND PROFILES ───────────────────────────────────────────────────────────┐
│ Plot ELF along non-equivalent nearest-neighbour bonds from an ELFCAR:         │
│   ./elf_bonds.py [ELFCAR] [--out PREFIX] [--nn-scale 1.2] [--npoints 200]     │
│                                                                               │
│ Reads ELFCAR directly (numpy + matplotlib only); groups bonds by              │
│ element pair + length; samples ELF centre-to-centre with PBC.                 │
│ Profiles BOTH the 1st and 2nd neighbour shells (dashed = 2nd; --shells 1     │
│ for 1st only).                                                                │
│                                                                               │
│ HONEST RENDERING: VASP's ELFCAR is the PSEUDO-ELF (no PAW reconstruction),   │
│ unphysical inside the augmentation spheres (RCORE, read from the POTCAR).    │
│ 1D: dotted inside the spheres, solid outside; a box reports outside-only     │
│ ⟨ELF⟩, ⟨|ELF'|⟩, ⟨ELF''⟩ per bond. 2D: semi-transparent gray discs mark      │
│ the spheres. Interpret bonding from the OUTSIDE region + shape only.         │
│ Output: PREFIX_elf_bonds.png/pdf  (PREFIX defaults to reduced formula,        │
│ e.g. MoSe2_elf_bonds.pdf).  ELF=0.5 electron-gas reference marked.            │
│                                                                               │
│ analyze.sh runs it when 02_scf/ELFCAR exists; GUI Results shows it too.       │
│ Note: VASP writes an all-zero ELFCAR for SOC / non-collinear runs;            │
│   the tool reports this instead of a blank plot — drop SOC for ELF.           │
└───────────────────────────────────────────────────────────────────────────────┘

┌─ CONVERGENCE TESTS ───────────────────────────────────────────────────────────┐
│ Range format (auto-steps):                                                    │
│   Convergence: Test k-points from 6x6x1 to 18x18x1    2D material           │
│                Test k-points from 6x6x6 to 12x12x12   3D material           │
│                ENCUT 400-600 eV                                               │
│                                                                               │
│ Explicit list:                                                                │
│   Convergence: Test k-points 6x6x3, 12x12x6, 18x18x9                       │
│                                                                               │
│ Generated directories: 00_convergence/kpoints/  and  00_convergence/encut/   │
│ Run:  ./run_convergence.sh   (workstation)                                    │
│       ./submit_convergence.sh (SLURM)                                        │
│ Plot: ./analyze_convergence.sh                                                │
│                                                                               │
│ The convergence POSCAR has atom 1 shifted by (0.01 0.02 0.03) Ang CARTESIAN  │
│ so the force on it is non-zero and its convergence is meaningful; all         │
│ production steps (01_relax, 02_scf, ...) keep the ORIGINAL positions.         │
│                                                                               │
│ Auto-selection (00_convergence/choose_params.py, run automatically):          │
│   smallest ENCUT / k-mesh whose successive force change on atom 1 stays      │
│   < 5 meV/Ang AND stress-diagonal change stays < 0.5 kbar; written to        │
│   00_convergence/converged_params.txt. Phase 2 (run_calculations.sh /        │
│   submit_calculations.sh / GUI) offers these as defaults; run non-           │
│   interactively (e.g. nohup, chained SCF) it uses them automatically.        │
│   Override thresholds:  choose_params.py --ftol 3 --ptol 0.2                 │
└───────────────────────────────────────────────────────────────────────────────┘

┌─ MPI / PARALLELIZATION ────────────────────────────────────────────────────────┐
│ Global settings:                                                              │
│   MPI: 128              Total MPI tasks (nodes × cores_per_node)             │
│   KPAR: 4               k-point parallelization                               │
│   NCORE: 8              Cores per orbital                                     │
│                                                                               │
│ Per-task overrides:                                                           │
│   SCF_KPAR: 4        BANDS_KPAR: 1       DOS_KPAR: 8                         │
│   RELAX_KPAR: 2      PHONONS_KPAR: 1                                          │
│   SCF_NCORE: 8       BANDS_NCORE: 16     DOS_NCORE: 8                        │
│   RELAX_NCORE: 8     PHONONS_NCORE: 8                                         │
│                                                                               │
│ Auto-defaults when not specified (N = total MPI tasks):                      │
│   SCF / DOS / relax  →  KPAR = floor(√N),  NCORE = N / KPAR                 │
│   bands              →  KPAR = 1,           NCORE = N                        │
│   DFPT               →  KPAR = 1, NCORE = 1  (forced: linear response/PEAD  │
│                          supports neither; overrides are ignored)            │
│   phonons            →  KPAR = 1,           NCORE = floor(√N)               │
│                                                                               │
│ Rule: KPAR × NCORE must divide N evenly. Agent enforces this automatically.  │
└───────────────────────────────────────────────────────────────────────────────┘

┌─ DFPT  (Born charges, dielectric & piezoelectric tensors) ────────────────────┐
│ Tasks: DFPT Born charges and dielectric                                       │
│                                                                               │
│ DFPT_EDIFF: 1E-8            electronic convergence (default 1E-8)             │
│                                                                               │
│ Route is chosen automatically:                                                │
│   PBE / PBEsol / LDA, no SOC → IBRION=8, LEPSILON=.TRUE.  (linear response)  │
│   SOC / HSE06 / R2SCAN       → IBRION=6, LCALCEPS=.TRUE.  (finite field;     │
│                                needs an insulating system, recent VASP 6)    │
│ KPAR and NCORE are forced to 1 — required by both routes.                     │
│ ISYM = 2 is enforced (symmetry imposed): a symmetry-off ISYM=0/-1 — e.g.      │
│ from a global INCAR override block meant for LOBSTER/NSCF — never leaks in.   │
│ Inputs come from 02_scf at run time (copy_from_scf.sh). If 02_scf was         │
│ converted in place for LOBSTER (ISYM=0/-1), its WAVECAR is skipped (k-mesh    │
│ mismatch) and DFPT starts from the still-symmetric CHGCAR; a warning is       │
│ printed if the CHGCAR itself may be from a symmetry-off SCF.                  │
│                                                                               │
│ Example instructions.txt:                                                     │
│     Project: GaAs                                                             │
│     Methods: PBE functional                                                   │
│     Tasks: relaxation, SCF, DFPT Born charges and dielectric                  │
│     MPI: 16                                                                   │
│                                                                               │
│ Outputs (06_dfpt):                                                            │
│   born_charges.txt   eps_inf, ionic contrib, total eps_0; Z* per element;    │
│                      acoustic-sum-rule check; piezoelectric e (C/m^2)        │
│   BORN               phonopy NAC file for LO-TO splitting                     │
└───────────────────────────────────────────────────────────────────────────────┘

┌─ PHONONS (phonopy) ────────────────────────────────────────────────────────────┐

│ PHONONS_DIM: 2 2 2          supercell dimensions                             │
│ PHONONS_MESH: 20 20 20      q-point mesh for thermal properties              │
│ PHONONS_DISP: 0.01          displacement amplitude (Å)                       │
│ PHONONS_BAND: G M K G A     high-symmetry path for phonon bands              │
│ PHONONS_NAC: FALSE          disable non-analytic correction (default: TRUE)  │
│                                                                               │
│ Note: DFPT Born charges are needed for NAC. Include DFPT task or set NAC off.│

┌─ LOBSTER  (COHP / COBI / COOP bonding analysis) ──────────────────────────────┐
│ Tasks: LOBSTER COHP/COBI analysis   (also: cohp, cobi, coop, bonding)         │
│                                                                               │
│ Adds 08_lobster: a symmetry-off (ISYM=0) NSCF reading 02_scf/CHGCAR, then     │
│ runs the lobster binary. ISYM=0 is REQUIRED -- a symmetrized k-mesh gives     │
│ unusable COHP/ICOHP (wrong sign, |Δ| > 100%). With SOC on, ISYM=-1 is set     │
│ instead (spin-orbit breaks time reversal). NBANDS is auto-set to >= the       │
│ LOBSTER basis-function count; LMAXMIX from the elements (6 f / 4 d / 2 sp).   │
│ Binary: $LOBSTER_BIN  (default lobster-5.1.0-OSX; ~ in site.env is expanded). │
│ lobsterin cohp/cobiGenerator range is structure-aware: it reaches every       │
│ atom's 2nd-neighbour shell (+10%, floor 4 Å, cap 6 Å), so 1st AND 2nd shell  │
│ bonds (e.g. metal-metal in rocksalt) are computed and plotted (dashed = 2nd). │
│                                                                               │
│ Example instructions.txt:                                                     │
│     Project: NaCl                                                             │
│     Methods: PBE functional                                                   │
│     Tasks: SCF, LOBSTER COHP/COBI analysis                                    │
│     MPI: 16                                                                   │
│                                                                               │
│ Run:        ./vasp-agent.py            (builds 02_scf + 08_lobster, runs both)│
│ Aggregate:  python3 lobster_postprocess.py   -> lobster_acohp_acobi.csv       │
└───────────────────────────────────────────────────────────────────────────────┘

│   ./ht-mp-scf.py --agent local  --mpi 16    workstation (sequential)          │

│                                                                               │
│ Options:                                                                      │
│   --agent local|slurm   which agent (prompts if omitted)                      │
